Introduction

The program SYSTRE is designed to analyze periodic nets as they arise in the study of (extended or non-molecular) crystal structures. The acronym stands for Symmetry, Structure and Refinement. SYSTRE uses a method called barycentric placement to determine the ideal (i.e., maximal embeddable) symmetry of a crystal net and to analyze its topological structure. It then generates a unique key for the topological type of the given net and uses this key to look up the structure in its built-in database. Users can also create their own databases.

SYSTRE also computes a refined embedding (geometric conformation) of the given net using a method based on simulated (physically incorrect) forces. The targeted embedding is one that minimizes the edge length variance (normalized to unit length) and maximizes the unit cell volume.


Installation

SYSTRE needs a Java runtime environment (JRE) of version 1.4.2 or higher to run. The most straightforward way to get a JRE is to download it from java.sun.com. You might already have a useable Java installation on your computer, though, because many programs use it.

Installing SYSTRE itself is very easy. Just download the file Systre.jar and put it anywhere you like, for example on your desktop.


Usage

The simplest way to start SYSTRE is to double-click on its icon. If that does not work, open a console window and type the command 

    java -jar Systre.jar

from the same folder that you put Systre.jar in.


File Formats

Here are some examples of legal input for SYSTRE. The conventional suffix for SYSTRE files is .cgd. You can cut and paste any of the following examples (or all of them) in a .cgd file using your favorite text editor (e.g. Notepad on Windows) and open it from SYSTRE.

The SYSTRE format allows comments, which start with a # character and continue until the end of the current line.


Example 1: Diamond, specified by space group, unit cell and positions.

# The type of data to expect
CRYSTAL
  # Structure id (optional, NAME is equivalent to ID)
  ID    diamond
  # The space group by name
  GROUP Fd-3m
  # Cell parameters: a b c alpha beta gamma
  CELL         2.3094 2.3094 2.3094  90.00000   90.00000   90.00000
  # Atom specification: name coordination x y z
  #   (decimal numbers or fractions can be used for coordinates)
  ATOM  1  4   5/8 5/8 5/8
# This ends the structure description
END

Example 2: Diamond, specified as a periodic graph.

PERIODIC_GRAPH
ID diamond
EDGES
  1 2 0 0 0
  1 2 1 0 0
  1 2 0 1 0
  1 2 0 0 1
END

Example 3: Edges can be given explicitly.

CRYSTAL
  NAME  real CdSO4
  GROUP Pmn21
  CELL  6.558 4.698 4.719 90.0 90.0 90.0
  # First kind of atom
  ATOM  1 4  0.0000  0.6657  0.7306
  # Edges can be specified by giving the names of the endpoints
  EDGE  1 2
  # or by giving a number for one and coordinates for the other. 
  EDGE  1    0.0000  0.1416  0.2500
  # or by giving coordinates for both
  EDGE  0.0000  0.6657  0.7306    0.0000  1.1416  1.2500
  # The Keyword EDGE may be missing on consecutive lines.
        1   -0.5000  0.8584  0.7500
  # Second kind of atom.
  ATOM  2 4  0.5000  0.8584  0.7500
  # Second group of edge specifications.
  EDGE  2    0.0000  0.6657  0.7306
        2    1.0000  0.6657  0.7306
        2    0.5000  1.3343  0.2306
        2    0.5000  0.3343  1.2306
END

Example 4: This example describes a more complicated graph and will take SYSTRE significantly longer to complete. It shows two situations in which quotes are required. 

CRYSTAL
  ID    LTN
  # Group Fd-3m with first origin choice instead of second
  #     (likewise, use ":R" for rhombohedral setting, where applicable)
  GROUP "Fd-3m:1"
  CELL  35.622 35.622 35.622 90.000 90.000 90.000
  ATOM
    # If the first field in a line starts with a letter, it is assumed
    # to be a keyword, so in this example, the atom names need to be quoted.
    "SI1" 4 0.3112 0.2500 0.3727
    "SI2" 4 0.4345 0.2481 0.3721
    "SI3" 4 0.3897 0.3285 0.4532
    "SI4" 4 0.5396 0.3394 0.3996
END

Troubleshooting

There are some quirks and limitations that you should be aware of when using SYSTRE.

Origin choice

If you specify a crystal as in the second example above, SYSTRE needs to compute edges based on nearest neighbors. This does not only depend on the correct cell parameters, but also on the origin choice. For those groups that have two conventional choices as according to the international tables, SYSTRE by default uses the second one. This can be changed in the 'Options' dialog.

Another way to make sure that the correct origin choice is used is to append :1 or :2, respectively, to the group name as in Example 4 above.

Rhombohedral groups

Hexagonal versus rhombohedral settings for rhombohedral groups are handled analogously to the above(append :H or :R, respectively).
